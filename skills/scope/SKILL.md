# Scope (/scope)

The first move of the spine: understand the system before you change it. No edits yet.

- Enter **plan mode**. Send an **`Explore`** scout to map the real entry path and find an existing
  pattern to reuse — it reads in a side context, so your main window stays sharp.
- Decide the **smallest change** that solves it. Name the file(s) you'll touch and why.
- Batch any clarifying questions now — before writing code, not after.
- If the change is non-trivial, draft a short plan, then **gate**: present it and get sign-off before editing.

Heuristic — *Scout, then step.* No edit until you can name the file you'll change and why.

Run the whole flow with **/lets-hack**.
